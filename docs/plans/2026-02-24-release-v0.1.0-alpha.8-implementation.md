### Task 3: GitHub Release

**Step 1: Prepare release notes file**

Create: `release-notes.md`

```markdown
**v0.1.0-alpha.8: Canonical Tool Taxonomy & Central Classifier**

This release marks the completion of the **M1.5 Finalize** milestone, introducing a unified, provider-aware tool taxonomy and central classifier. This ensures stable tool matching and classification across different AI providers.

#### Key Features

- **Unified Tool Classification**: A centralized classification engine featuring exact, pattern-based, and heuristic matching for tools across multiple providers.
- **Provider-Aware YAML Registry**: A hot-reloadable `configs/tool_registry.yaml` defining tools and their categories for Anthropic, OpenAI, and custom providers.
- **Enhanced Commandments Engine**: New `tool_category`, `tool_effect`, and `tool_name` match fields for more robust and provider-agnostic rules.
- **CLI Tool Introspection**: A new `crabwise classify` command for dry-running classifier resolution and debugging taxonomy matches.
- **Atomic Dual Reload**: SIGHUP handlers now atomically swap both commandments and the tool registry, ensuring consistent runtime behavior.
- **Improved Observability**: New `unclassified_tool_count` in daemon status to track and mitigate taxonomy drift.
```

**Step 2: Create GitHub release**

Run: `gh release create v0.1.0-alpha.8 --title "v0.1.0-alpha.8: Canonical Tool Taxonomy & Central Classifier" --notes-file release-notes.md`
Expected: Release created and URL returned.

**Step 3: Clean up**

Run: `rm release-notes.md`
Expected: File removed.

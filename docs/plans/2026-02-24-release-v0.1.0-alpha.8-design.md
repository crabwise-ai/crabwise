# Design: Release v0.1.0-alpha.8

## Overview
This design outlines the process and content for the `v0.1.0-alpha.8` release of Crabwise. This release focuses on the finalization of the M1.5 milestone: Canonical Tool Taxonomy & Central Classifier.

## Release Metadata
- **Version**: `v0.1.0-alpha.8`
- **Release Type**: Alpha
- **Primary Milestone**: M1.5 Finalize

## Release Description (User-Friendly)

**v0.1.0-alpha.8: Canonical Tool Taxonomy & Central Classifier**

This release marks the completion of the **M1.5 Finalize** milestone, introducing a unified, provider-aware tool taxonomy and central classifier. This ensures stable tool matching and classification across different AI providers.

### Key Features

- **Unified Tool Classification**: A centralized classification engine featuring exact, pattern-based, and heuristic matching for tools across multiple providers.
- **Provider-Aware YAML Registry**: A hot-reloadable `configs/tool_registry.yaml` defining tools and their categories for Anthropic, OpenAI, and custom providers.
- **Enhanced Commandments Engine**: New `tool_category`, `tool_effect`, and `tool_name` match fields for more robust and provider-agnostic rules.
- **CLI Tool Introspection**: A new `crabwise classify` command for dry-running classifier resolution and debugging taxonomy matches.
- **Atomic Dual Reload**: SIGHUP handlers now atomically swap both commandments and the tool registry, ensuring consistent runtime behavior.
- **Improved Observability**: New `unclassified_tool_count` in daemon status to track and mitigate taxonomy drift.

---

## Release Process

### 1. Verification
Before creating the release, the following checks must be performed:
- [ ] Run `go test ./...` to ensure all tests pass.
- [ ] Verify `crabwise classify` works as expected.
- [ ] Ensure the tool registry YAML is valid.

### 2. Git Tagging
Create a new git tag:
```bash
git tag -a v0.1.0-alpha.8 -m "v0.1.0-alpha.8: Canonical Tool Taxonomy & Central Classifier"
git push origin v0.1.0-alpha.8
```

### 3. GitHub Release Creation
Create a GitHub release using the `gh` CLI:
```bash
gh release create v0.1.0-alpha.8 --title "v0.1.0-alpha.8: Canonical Tool Taxonomy & Central Classifier" --notes-file release-notes.md
```
*(Note: `release-notes.md` will contain the user-friendly description above)*

## Rollback Plan
If issues are discovered after release:
- Delete the tag: `git tag -d v0.1.0-alpha.8 && git push origin :refs/tags/v0.1.0-alpha.8`
- Delete the GitHub release via `gh release delete v0.1.0-alpha.8`.
